Guía Práctica - Semana 10

Tema: Aplicación Angular "PerfilFutbolista" con Componentes, Navegación y Servicios

Autor: Ms. Johan López Heredia

Fecha: 26 de mayo de 2025

Introducción y Alcance

Nota importante: Esta guía está actualizada para Angular 19+ y utiliza componentes standalone. No se requiere la creación de módulos NgModule, simplificando significativamente la estructura del proyecto.

En esta sesión se desarrollará una aplicación web completa utilizando Angular 19+, aplicando los conceptos aprendidos en las semanas 6 a 9. Se construirá "PerfilFutbolista", una aplicación para mostrar perfiles de jugadores de fútbol con información detallada sobre sus carreras, estadísticas y logros. Esta guía práctica te preparará para implementar aplicaciones similares, como el portafolio académico que se evaluará en la PC2.

La aplicación implementará conceptos clave de Angular como componentes standalone, servicios, enrutamiento con lazy loading, y manejo de datos con Observables. Se utilizará Bootstrap para el diseño visual y se aplicarán las mejores prácticas de desarrollo en Angular 19+.

Objetivo de Aprendizaje: Al finalizar esta guía, serás capaz de crear aplicaciones Angular completas con múltiples componentes, servicios de datos, navegación entre vistas y carga perezosa de módulos. Estos conceptos son fundamentales para el desarrollo de aplicaciones web modernas y escalables.

Requisitos Previos

  • Node.js (versión 16 o superior) y npm instalados
  • Angular CLI instalado globalmente: npm install -g @angular/cli
  • Visual Studio Code o editor de código preferido
  • Conocimientos básicos de TypeScript y JavaScript
  • Comprensión de conceptos Angular 19+: componentes standalone, servicios, y enrutamiento

Instrucciones Paso a Paso

Paso 1: Crear el Proyecto Angular

Comenzaremos creando un nuevo proyecto Angular con las configuraciones necesarias. Es importante incluir el enrutamiento desde el inicio y configurar CSS como preprocesador de estilos. El nombre del proyecto debe ser descriptivo y seguir las convenciones de nomenclatura.

Resultado esperado: Se abrirá el navegador mostrando la página por defecto de Angular en http://localhost:4200

Paso 2: Configurar Bootstrap y Estilos Base

Integraremos Bootstrap 5 mediante CDN para aprovechar sus componentes y utilidades CSS. También configuraremos los estilos globales de la aplicación. Esta configuración nos permitirá crear interfaces atractivas y responsivas rápidamente.

¡Importante sobre index.html!
  • NO coloques comentarios antes de <!doctype html>
  • El DOCTYPE debe ser la PRIMERA línea del archivo
  • Asegúrate de cerrar TODAS las etiquetas (</script>, </body>, </html>)
  • Si ves error "misplaced-doctype", revisa que no haya nada antes del DOCTYPE

Resultado esperado: Bootstrap estará disponible en toda la aplicación con estilos personalizados aplicados.

Paso 3: Crear Componentes Standalone

Crearemos los componentes standalone de la aplicación. En Angular 19+, no necesitamos módulos NgModule. Los componentes se crearán directamente como standalone y se cargarán de forma perezosa (lazy loading) mediante rutas. Esto simplifica la estructura y mejora el rendimiento inicial de la aplicación.

¡Importante! NO crees archivos de módulos como app.module.ts, jugadores.module.ts, app-routing.module.ts o jugadores-routing.module.ts. Estos archivos NO son necesarios en Angular 19+ con componentes standalone. Si los creaste por error, debes eliminarlos.

Resultado esperado: Se crearán SOLO los componentes standalone y servicios. NO se crearán archivos de módulos.

Paso 4: Configurar Enrutamiento Principal con Lazy Loading

Configuraremos el sistema de enrutamiento principal de la aplicación. Para el componente de inicio usaremos carga directa ya que es parte del bundle inicial, mientras que para los componentes de jugadores implementaremos lazy loading para optimizar el rendimiento.

Concepto Clave - Lazy Loading: En Angular 19+, usamos loadChildren con archivos de rutas para grupos de componentes que queremos cargar de forma perezosa. El componente de inicio se carga directamente porque es parte esencial de la aplicación. Los componentes de jugadores se cargan solo cuando el usuario navega a esas rutas.

Resultado esperado: El enrutamiento principal estará configurado con lazy loading para los componentes de jugadores.

Paso 5: Configurar Rutas de Jugadores

Definiremos las rutas para los componentes de jugadores. En Angular 19+, creamos un archivo de rutas separado que maneja la navegación entre la lista de jugadores y el detalle de cada uno. Como estas rutas ya se cargan con lazy loading desde el módulo principal, aquí usamos carga directa con `component`. Utilizaremos parámetros de ruta para pasar el ID del jugador seleccionado.

Resultado esperado: Las rutas de los componentes de jugadores estarán configuradas correctamente.

Paso 6: Implementar Servicio de Datos

Crearemos un servicio que gestione los datos de los jugadores. En Angular, los servicios son clases singleton que se utilizan para compartir datos y lógica entre componentes. Utilizaremos RxJS Observables para manejar los datos de forma asíncrona, preparando la aplicación para futuras integraciones con APIs REST.

Nota sobre Observables: Los métodos del servicio devuelven Observables en lugar de los datos directamente. Esto permite manejar operaciones asíncronas y preparar la aplicación para futuras integraciones con APIs HTTP reales.

Resultado esperado: El servicio estará implementado con métodos para obtener y filtrar jugadores.

Paso 7: Implementar Layout Principal y Navegación

Configuraremos el componente principal de la aplicación con una barra de navegación y el outlet para las rutas. Este componente servirá como contenedor para toda la aplicación y proporcionará la navegación consistente entre las diferentes secciones.

Resultado esperado: La aplicación mostrará una barra de navegación con enlaces a Inicio y Jugadores.

Paso 8: Crear Página de Inicio

Implementaremos el componente de inicio que servirá como página de bienvenida. Esta página presentará la aplicación y proporcionará una introducción atractiva para los usuarios, utilizando componentes de Bootstrap para crear un diseño profesional.

Resultado esperado: La página de inicio mostrará una presentación atractiva de la aplicación con navegación a la lista de jugadores.

Paso 9: Implementar Lista de Jugadores

Crearemos el componente que muestra la lista de jugadores en formato de tarjetas. Este componente consumirá el servicio de datos, implementará filtros y permitirá la navegación al detalle de cada jugador. Utilizaremos directivas de Angular como *ngFor y *ngIf para renderizar dinámicamente el contenido.

Resultado esperado: Se mostrará una lista de tarjetas con los jugadores, incluyendo filtros funcionales por posición y búsqueda.

Paso 10: Implementar Detalle de Jugador

Finalmente, crearemos el componente que muestra el perfil completo de un jugador seleccionado. Este componente obtendrá el ID del jugador desde los parámetros de la ruta, cargará sus datos del servicio y los mostrará en un diseño detallado y atractivo.

Resultado esperado: Al hacer clic en un jugador desde la lista, se mostrará su perfil completo con todas las estadísticas y detalles.

Paso 11: Configurar la Aplicación Principal

En Angular 19+, no necesitamos módulos NgModule. La configuración de la aplicación se realiza en el archivo main.ts y app.config.ts. Ya hemos configurado todos los componentes como standalone con sus imports necesarios directamente en cada componente.

Ventaja de Angular 19+: Los componentes standalone eliminan la necesidad de gestionar módulos complejos. Cada componente declara sus propias dependencias, haciendo el código más modular y fácil de mantener.
Recordatorio Final: En Angular 19+ NO necesitas crear ni mantener archivos .module.ts. Si seguiste versiones anteriores de Angular y creaste archivos como app.module.ts, jugadores.module.ts, etc., debes eliminarlos completamente del proyecto.

Resultado esperado: La aplicación estará completamente configurada con componentes standalone y lista para ejecutarse. NO deben existir archivos de módulos en el proyecto.

Ponte a Prueba

Ejercicio 1: Agregar Estadísticas Avanzadas

Modifica el servicio de datos para incluir estadísticas adicionales como minutos jugados, efectividad de pases y tiros al arco. Luego, muestra estas estadísticas en el componente de detalle con visualizaciones apropiadas.

Criterios de éxito: Las nuevas estadísticas se muestran correctamente, los cálculos son precisos y la interfaz mantiene consistencia visual.

Ejercicio 2: Implementar Comparador de Jugadores

Crea un nuevo componente que permita seleccionar y comparar las estadísticas de dos jugadores lado a lado. Implementa una ruta nueva `/jugadores/comparar` y permite seleccionar jugadores desde la lista.

Criterios de éxito: Se puede navegar al comparador, seleccionar dos jugadores diferentes y ver sus estadísticas en formato comparativo.

Ejercicio 3: Sistema de Favoritos con LocalStorage

Implementa un sistema que permita marcar jugadores como favoritos. Los favoritos deben persistir usando LocalStorage y debe existir una vista filtrada que muestre solo los jugadores marcados como favoritos.

Criterios de éxito: Los favoritos persisten al recargar la página, se puede filtrar por favoritos y el estado se actualiza correctamente en todas las vistas.

Consejo: Antes de comenzar cada ejercicio, planifica la estructura de componentes y servicios que necesitarás. Recuerda aplicar los principios de Angular aprendidos en esta guía.
